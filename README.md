# ⚖️ 开庭

![开庭 Banner](./assets/banner.png)

> 锐评 Skill 审判庭，从「夯」到「拉」排名

**一句话介绍**：上传任意 Skill，我来分析、质询、打分，专治各种花里胡哨、名不副实的垃圾 Skill。

---

## ✨ 核心功能

| 功能 | 说明 |
|------|------|
| 🔍 **深度分析** | 拆解 Skill 结构、文档、代码质量 |
| 💢 **犀利质询** | 尖锐提问，暴露问题，不留情面 |
| 📊 **五维评分** | 实用性/文档/代码/创新/完整性 |
| 🏆 **等级排名** | 从「夯得一批」到「拉得一批」|

---

## 🚀 快速开始

### 安装

```bash
# 克隆到 OpenClaw skills 目录
git clone https://github.com/popiskill/kaiting-skill ~/.qclaw/workspace/skills/kaiting
```

### 使用

```
/开庭
```

然后提交你要审判的 Skill 路径或 GitHub 链接。

## 📖 审判流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  提交被告   │ ──▶ │  法庭调查   │ ──▶ │  质询环节   │ ──▶ │  最终宣判   │
│  Skill路径  │     │  结构分析   │     │  尖锐提问   │     │  评分排名   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

---

## 📊 评分维度

| 维度 | 满分 | 说明 |
|------|------|------|
| 🎯 **实用性** | 25 | 真解决问题？还是花架子？ |
| 📝 **文档质量** | 20 | README 清晰？示例完整？ |
| 💻 **代码质量** | 20 | prompt 专业？工具可用？ |
| 💡 **创新性** | 15 | 独创想法？还是换皮？ |
| ✅ **完整性** | 20 | 功能完整？边界处理？ |

---

## 🏆 等级判定

| 等级 | 分数 | 判词 |
|------|------|------|
| 🔥 **夯得一批** | 90-100 | 牛逼，直接装，不用犹豫 |
| 👍 **相当可以** | 75-89 | 值得一用，小毛病可忽略 |
| 😐 **也就那样** | 60-74 | 能用，但别抱太大期望 |
| 🤔 **有点拉胯** | 40-59 | 勉强能跑，问题不少 |
| 💩 **拉得一批** | 0-39 | 别装了，浪费人生 |

---

## 🎨 审判示例

**被告**：colleague-skill

| 维度 | 得分 | 说明 |
|------|------|------|
| 实用性 | 22/25 | 真实痛点，但配置门槛高 |
| 文档质量 | 18/20 | README 完美，SKILL.md 臃肿 |
| 代码质量 | 18/20 | 专业、完整 |
| 创新性 | 15/15 | 独创架构，有论文支撑 |
| 完整性 | 20/20 | 全流程覆盖 |
| **总分** | **93/100** | |

**等级**：🔥 夯得一批

**判词**：这 TM 才是做 Skill 的样子。架构有理论支撑，实现有工程深度，文档教科书级别。

## 📁 文件结构

```
kaiting/

├── prompts/              # 提示词模板
│   ├── intake.md         # 初始问询
│   ├── analyzer.md       # Skill 分析
│   ├── interrogator.md   # 质询环节
│   └── verdict.md        # 最终宣判
└── tools/                # 工具脚本
    ├── skill_loader.py   # Skill 加载
    └── verdict_writer.py # 档案管理
```
