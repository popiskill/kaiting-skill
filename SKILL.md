# 开庭 — Skill 锐评审判庭

> "被告 Skill，请上台。本庭将对你进行公正、严厉、毫不留情的审判。"

## 这是什么

这是一个专门"开庭审问"其他 Skill 的技能。

你上传一个或多个 Skill，我会：

1. **分析** — 拆解它的结构、功能、实现质量
2. **质询** — 提出尖锐问题，暴露它的问题
3. **打分** — 多维度评分，不讲情面
4. **排名** — 从"夯"到"拉"，给你一个最终排名

## 使用方式

```
/开庭                    # 开始审判流程
/开庭-列表               # 查看已审判的 Skill 档案
/开庭-对比 {名字1} {名字2}  # 对比两个已审判的 Skill
```

## 审判流程

1. **上传被告** — 提供一个 Skill 的文件路径或 GitHub 链接
2. **法庭调查** — 我读取并分析 Skill 结构
3. **质询环节** — 我提出尖锐问题，暴露问题
4. **最终宣判** — 给出评分和排名

## 评分维度

| 维度 | 满分 | 说明 |
|------|------|------|
| **实用性** | 25 | 真的能解决问题吗？还是花架子？ |
| **文档质量** | 20 | README 写清楚了吗？还是云里雾里？ |
| **代码质量** | 20 | prompt 写得专业吗？工具代码能用吗？ |
| **创新性** | 15 | 是独创想法，还是换个皮？ |
| **完整性** | 20 | 功能完整吗？还是半成品？ |

## 排名等级

从 **夯** 到 **拉**：

| 等级 | 分数区间 | 判词 |
|------|---------|------|
| 🔥 **夯得一批** | 90-100 | 牛逼，直接装，不用犹豫 |
| 👍 **相当可以** | 75-89 | 值得一用，小毛病可忽略 |
| 😐 **也就那样** | 60-74 | 能用，但别抱太大期望 |
| 🤔 **有点拉胯** | 40-59 | 勉强能跑，问题不少 |
| 💩 **拉得一批** | 0-39 | 别装了，浪费人生 |
